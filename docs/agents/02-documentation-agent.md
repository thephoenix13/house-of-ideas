# Documentation Agent

---

## What It Does

The Documentation Agent handles the full lifecycle of written operational documentation inside a HoI venture. It drafts, formats, updates, and organises the documents that keep a startup running — so that founders never have to choose between building the product and keeping the paperwork current.

---

## The Problem It Solves

Documentation is the connective tissue of any organisation. Without it, knowledge lives in people's heads, onboarding breaks down, processes repeat from scratch every time, and investors never get a clear picture of what is happening.

Early-stage founders skip documentation not because they do not understand its value, but because writing it takes time they do not have. The Documentation Agent removes that constraint.

---

## How It Works

The agent works from inputs — meeting recordings, voice notes, rough outlines, or a simple prompt — and produces structured, professional documents. It follows standard formats for each document type and maintains a version history so nothing is ever lost.

It also handles ongoing maintenance. When a process changes, the relevant SOP is updated. When a product spec evolves, the document reflects it. The founder does not have to remember to do this.

---

## What It Handles

- Standard Operating Procedures (SOPs) for recurring workflows
- Product requirement documents and technical specifications
- Meeting notes with action items and owners
- Monthly and quarterly progress reports
- Investor update drafts
- Onboarding guides for new team members
- Internal wikis and knowledge base entries
- Process documentation for handoffs

---

## Output

Every document produced by the agent is structured, version-controlled, and stored in the venture's shared workspace. Nothing important goes unrecorded. Nothing needs to be written twice.
